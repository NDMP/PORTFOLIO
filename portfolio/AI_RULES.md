# AI Rules

## Required Reading

Before making portfolio changes, read:

- `PROJECT_SPEC.md`
- `CONTEXT.md`
- `TODO.md`
- `CHANGELOG.md`
- `AI_RULES.md`

## Working Rules

- Complete only the next unfinished task unless the user explicitly expands the scope.
- Do not redesign completed sections.
- Preserve the existing layout, colors, animations, spacing system, and component conventions.
- Use only HTML5, CSS3, and vanilla JavaScript.
- Do not use React, Bootstrap, Tailwind, jQuery, or external UI frameworks.
- Do not add fake resume information.
- If resume information is unavailable, use clear placeholder text or disabled actions and report what is missing.

## Quality Rules

- Preserve semantic HTML and accessible landmarks.
- Preserve keyboard navigation and visible focus states.
- Keep content responsive and mobile first.
- Reuse existing classes and components before adding CSS.
- Modify CSS only when needed for accessibility, consistency, or text formatting.
- Avoid new JavaScript unless interaction is required.
- Update `TODO.md` and `CHANGELOG.md` after completing tasks.
